
# Vue 代码审查测试项目

本项目用于测试 AI 代码审查能力，包含故意引入的代码问题。

## 项目结构

```
vue-code-review-demo/
├── src/                          # 初始代码版本（有问题的代码）
│   ├── main.js                   # 入口文件
│   ├── App.vue                   # 主应用组件
│   ├── components/
│   │   ├── UserList.vue          # 用户列表组件
│   │   └── UserProfile.vue       # 用户详情组件
│   ├── views/
│   │   └── Dashboard.vue         # 仪表盘视图
│   ├── api/
│   │   └── users.js              # API 调用
│   ├── utils/
│   │   └── helpers.js            # 工具函数
│   │   └── helpers-original.js   # 原始问题代码
│   │   └── helpers-fixed.js      # 修复后代码
│   ├── store/
│   │   └── userStore.js          # 状态管理
│   └── router/
│       └── index.js              # 路由配置
│
└── src-fixed/                    # 修复后的代码版本
    ├── main.js
    ├── App.vue
    ├── components/
    ├── views/
    ├── api/
    ├── utils/
    ├── store/
    └── router/
```

## 代码问题清单

### 1. main.js - 全局配置问题

| 问题 | 描述 | 严重程度 |
|------|------|----------|
| 硬编码配置 | baseURL 和 timeout 硬编码在代码中 | 中 |
| 错误信息全局暴露 | window.globalError 暴露敏感错误信息 | 高 |
| 缺少环境变量 | 没有使用环境变量管理配置 | 中 |
| 错误处理不统一 | axios 拦截器直接 reject，未标准化错误 | 中 |

### 2. router/index.js - 路由安全

| 问题 | 描述 | 严重程度 |
|------|------|----------|
| 无路由守卫 | 没有身份验证和授权检查 | 高 |
| 缺少 404 路由 | 没有处理未知路由 | 低 |
| 无错误处理 | 路由切换失败无提示 | 中 |

### 3. api/users.js - API 层问题

| 问题 | 描述 | 严重程度 |
|------|------|----------|
| 无参数验证 | 传入的参数未验证 | 中 |
| 无错误处理 | Promise reject 但没有统一处理 | 中 |
| XSS 风险 | searchUsers 直接拼接用户输入到 URL | 高 |
| 无缓存机制 | 重复请求相同数据 | 中 |
| 缺少类型定义 | 返回值类型不明确 | 低 |

### 4. utils/helpers.js - 工具函数问题

| 问题 | 描述 | 严重程度 |
|------|------|----------|
| deepClone 错误实现 | JSON.parse/stringify 无法处理循环引用和特殊类型 | 高 |
| debounce 实现不完整 | 缺少 immediate 参数支持 | 中 |
| throttle 实现错误 | 第一次调用不会立即执行 | 高 |
| generateId 不安全 | Math.random() 不是加密安全的随机数 | 中 |
| storage 无错误处理 | 可能抛出异常导致程序崩溃 | 中 |
| 缺少 XSS 防护 | 没有 HTML 转义函数 | 高 |

### 5. store/userStore.js - 状态管理问题

| 问题 | 描述 | 严重程度 |
|------|------|----------|
| 竞态条件 | 多个组件同时修改状态可能导致不一致 | 高 |
| 无响应式 | 使用普通变量而非 Vue 响应式系统 | 高 |
| 无缓存机制 | 重复请求相同数据 | 中 |
| 数据直接引用 | 返回原始数据引用，可被外部修改 | 中 |

### 6. components/UserList.vue - 组件问题

| 问题 | 描述 | 严重程度 |
|------|------|----------|
| XSS 漏洞 | 直接使用 v-html 或未转义用户输入 | 高 |
| 缺少权限检查 | 未验证用户权限 | 中 |
| 无确认对话框 | 删除操作没有二次确认 | 中 |
| URL 注入风险 | searchQuery 直接拼接到 URL | 高 |
| 内存泄漏 | 定时器未清理 | 中 |

### 7. components/UserProfile.vue - 性能问题

| 问题 | 描述 | 严重程度 |
|------|------|----------|
| 重复请求 | 同一个用户发起多次请求 | 中 |
| 内存泄漏 | setInterval 未清理 | 高 |
| 缺少路由监听 | params.id 变化时未重新加载 | 中 |
| 密码提示不安全 | prompt() 明文输入密码 | 高 |
| 无加载状态缓存 | 切换用户时闪烁 | 低 |

### 8. views/Dashboard.vue - 安全问题

| 问题 | 描述 | 严重程度 |
|------|------|----------|
| 数据外泄 | exportData 发送到外部服务器 | 严重 |
| 无 CSRF 防护 | 导出请求无 token | 高 |
| 敏感数据暴露 | 直接导出所有数据 | 中 |
| 无导出确认 | 误操作风险 | 低 |

## 问题类型统计

| 类别 | 数量 | 严重程度分布 |
|------|------|--------------|
| 安全漏洞 | 8 | 2 严重, 4 高, 2 中 |
| 性能问题 | 4 | 1 高, 2 中, 1 低 |
| 代码质量 | 6 | 2 高, 3 中, 1 低 |
| 最佳实践 | 5 | 2 高, 2 中, 1 低 |

## 审查维度

1. **安全性**: XSS、CSRF、注入、敏感信息泄露
2. **性能**: 内存泄漏、重复请求、缓存策略
3. **代码质量**: 错误处理、代码复用、可维护性
4. **最佳实践**: Vue 3 组合式 API、响应式、类型安全

## 使用方式

### 查看原始问题代码
```bash
cat src/utils/helpers.js
```

### 查看修复后代码
```bash
cat src-fixed/utils/helpers.js
```

### 对比两个版本
```bash
diff src/utils/helpers.js src-fixed/utils/helpers.js
```
